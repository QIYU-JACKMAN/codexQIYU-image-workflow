# 电商自定分批

保持内联执行，不使用子代理。

## 决策优先级

严格使用 `用户指令 > 角色定位 > 默认电商规则`。用户指定的屏数、顺序、文案、图片编号、风格和禁用内容不能被默认规则覆盖。

输入包括自由指令、0-5 张商品图、可选参考图、语言、画幅、分辨率和质量。商品图先独立分析并请求确认，参考图随后单独收集和分析。

商品分析后读取 `custom-batch-presets.md`，根据品类和用途推荐一个规划角色以及 1-3 个视觉风格，让客户确认、替换、组合或选择自定义。默认角色是电商视觉总监。

指令有明确屏数时照做；没有时按列出的画面任务推断 1-20 屏。不同位置出现冲突屏数时，以用户高优先级指令为准并指出冲突。

## 执行

1. 定位插件根目录并读取公共参考，包括 `gpt-image-2-prompt-optimizer.md`。
2. 按“商品分析 → 角色/风格确认 → 自由指令 → 可选参考图确认”的顺序收集信息，创建 `custom-batch` 项目并为每屏分配一个主要目标。
3. 每屏保存从自由指令和角色定位中提取的原始需求，用内置优化器生成完整提示词，保留用户指定的 `image N` 映射，禁止引用不存在的图片。
4. 分批展示每屏的原需求、优化提示词、图片映射、不可改动项和补充内容，请客户批准或提出单点修改。未批准的任务保持 `pending`。
5. 所有启用任务通过客户校验后，令 `prompt` 等于已审阅的优化提示词并标记 `prompt_review.status=approved`，写入未批准计划并运行公共校验器。
6. 展示推断屏数、逐屏目标、图片映射、参数和目录；允许用户修改、排序或禁用任务。
7. 明确确认完整计划后批准，再优先使用 Codex 内置图片能力，否则运行公共 CLI 执行器。
8. 检查套图统一性、单屏约束和每屏指令覆盖率，只重做失败屏。
